# CommandCabin 截图功能设计

## 背景

CommandCabin 目前是 Windows-first Electron 启动器，已有全局快捷键、设置页、搜索命令、剪贴板能力、主进程 IPC 和独立窗口创建能力。截图功能应复用这些基础设施，不引入第二套桌面技术栈。

目标是实现熟悉的聊天式截图体验：快速框选、轻量标注、复制到剪贴板、保存文件、文字识别和贴图钉住。第一版不做滚动截图。

## 范围

第一版包含：

- 独立截图快捷键，默认 `Ctrl+Alt+A`，可在设置页自定义。
- 搜索命令入口：截图、3 秒延时截图、5 秒延时截图、文字识别截图。
- 启动截图前自动隐藏 CommandCabin 主窗口，避免截到自身。
- 全显示器统一截图层，支持跨屏框选。
- 框选时显示尺寸提示和光标放大镜。
- 选区下方浮动工具条，以图标按钮为主。
- 轻量标注：矩形、圆形、箭头、画笔、文字、马赛克。
- 标注样式：颜色、粗细、字号。
- 撤销和重做。
- 默认完成动作：点击完成、按 Enter、双击鼠标左键都会复制截图到剪贴板并关闭截图层。
- 保存按钮弹出系统保存对话框，默认 PNG，另支持 JPG。
- OCR 按钮手动触发本地离线文字识别，支持简体中文、繁体中文、英文。
- 钉住按钮创建置顶贴图窗口，支持拖拽、缩放、关闭。
- 取消入口：Esc、右键或取消按钮关闭截图层，不修改剪贴板。

第一版不包含：

- 滚动截图。
- 自动保存目录和自动保存规则。
- 标注对象级二次编辑、移动、缩放或删除。
- 贴图透明度、右键菜单和会话恢复。
- 云端 OCR 或上传图片识别。

## 技术方向

采用 Electron 原生截图层加少量 Windows 本地能力桥接。

- 主进程负责截图生命周期、快捷键注册、窗口创建、剪贴板、保存对话框、OCR 桥接。
- Renderer 使用 React 和 Canvas 实现选区、工具条、标注和 OCR 结果面板。
- 贴图使用独立置顶窗口，与截图 overlay 解耦。
- OCR 优先使用 Windows 本地离线能力；如果系统能力或语言包缺失，界面给出明确提示。

不采用独立 WinUI/Win32 截图模块，也不调用系统截图工具作为主流程。这样可以保持代码和发布链路集中在现有 Electron 架构内。

## 模块设计

### Screenshot Controller

主进程控制器负责：

- 注册默认截图快捷键 `Ctrl+Alt+A`。
- 响应设置页里的截图快捷键变更。
- 处理快捷键冲突和注册失败，保存失败时回滚到原设置。
- 注册截图相关搜索命令。
- 处理普通截图、3 秒延时截图、5 秒延时截图和文字识别截图入口。
- 截图启动前隐藏 CommandCabin 主窗口。
- 创建和关闭截图 overlay。
- 管理截图 IPC：复制、保存、钉住、OCR。

### Multi-display Capture Overlay

截图 overlay 负责：

- 在所有显示器上进入统一截图状态。
- 建立虚拟桌面坐标系，支持负坐标副屏和不同缩放比例。
- 捕获每个显示器的当前画面，拼成可框选的截图背景。
- 支持跨屏拖拽框选。
- 在多屏或 DPI 异常时回退到当前鼠标所在显示器，不让截图功能完全不可用。

### Capture Canvas And Annotation Engine

Canvas 层负责：

- 鼠标拖拽选区。
- 选区尺寸提示。
- 光标附近放大镜。
- 熟悉的轻量标注工具：矩形、圆形、箭头、画笔、文字、马赛克。
- 工具样式：颜色、线宽、字号。
- 操作栈撤销和重做。
- 输出时合成背景、裁剪区域和标注层。

标注对象画完后不支持复杂二次编辑。用户需要修改时使用撤销/重做重新绘制。

### Output Pipeline

输出管线负责：

- 点击完成、Enter、双击左键复制当前截图结果到剪贴板并关闭截图层。
- 保存按钮调用系统保存对话框。
- 保存格式支持 PNG 和 JPG，默认 PNG。
- 复制失败时保留 overlay 并提示错误。
- 保存失败或取消保存时保留 overlay，不丢失当前截图状态。

### Pinned Image Windows

贴图模块负责：

- 从当前选区生成置顶贴图窗口。
- 支持拖拽移动。
- 支持边缘或角落缩放。
- 支持关闭。
- 贴图窗口为临时状态，应用退出后不恢复。

贴图窗口不参与下一次截图 overlay 的状态管理。

### Local OCR Bridge

OCR 模块负责：

- 用户点击 OCR 按钮后识别当前选区。
- 默认不自动 OCR，避免拖慢普通截图流程。
- 支持简体中文、繁体中文、英文。
- 优先使用 Windows 本地离线 OCR 能力。
- 系统缺少 OCR 能力或语言包时，OCR 面板显示明确提示。
- OCR 结果显示在小面板中，支持一键复制全部文字。
- OCR 失败不影响截图复制、保存和贴图。

## 用户流程

### 普通截图

1. 用户按 `Ctrl+Alt+A` 或搜索“截图”。
2. CommandCabin 隐藏主窗口。
3. 截图 overlay 覆盖所有显示器。
4. 用户拖拽框选区域。
5. 选区旁显示尺寸，光标附近显示放大镜。
6. 用户可直接完成，也可使用工具条标注。
7. 完成按钮、Enter 或双击左键复制截图到剪贴板并关闭 overlay。

### 延时截图

1. 用户搜索“3 秒延时截图”或“5 秒延时截图”。
2. CommandCabin 隐藏主窗口。
3. 等待倒计时结束。
4. 进入普通截图 overlay。

第一版倒计时做轻量实现，不提供自定义秒数。

### 文字识别

1. 用户框选区域。
2. 用户点击 OCR 按钮。
3. 本地 OCR 识别选区。
4. 识别结果显示在小面板中。
5. 用户可复制全部文字，也可继续保存、标注、贴图或完成截图。

### 贴图钉住

1. 用户框选区域。
2. 用户点击钉住按钮。
3. 当前选区生成置顶贴图窗口。
4. 用户可以拖拽、缩放或关闭贴图。
5. 截图 overlay 可以关闭，贴图窗口继续存在。

## 设置与命令

设置页新增截图设置区域：

- 截图快捷键，默认 `Ctrl+Alt+A`。
- 快捷键录入体验复用现有热键设置。
- 注册冲突或失败时显示错误并回滚。

搜索命令新增：

- 截图。
- 3 秒延时截图。
- 5 秒延时截图。
- 文字识别截图。

第一版不新增自动保存设置。

## 错误处理

- 截图启动失败：显示可理解错误，不影响主启动器继续使用。
- 快捷键注册失败：保存失败并回滚到旧快捷键。
- 截图捕获失败：提示失败，恢复主窗口可用状态。
- 多屏坐标异常：回退到当前鼠标所在显示器。
- 剪贴板写入失败：保留 overlay，提示用户可改用保存。
- 保存失败：保留 overlay，不丢失当前截图。
- OCR 不可用：OCR 面板显示本地能力或语言包缺失提示。
- OCR 识别失败：只影响 OCR 面板，不影响截图其他输出。
- 贴图窗口创建失败：提示失败，截图 overlay 保留。

## 测试策略

自动化测试：

- 主进程：截图快捷键注册、设置保存/回滚、搜索命令触发、延时计时、窗口创建/关闭、剪贴板 IPC、保存 IPC、OCR IPC。
- Renderer：选区状态机、键盘快捷键、工具选择、撤销/重做、工具条按钮、错误提示。
- Canvas：标注命令绘制、马赛克区域、裁剪坐标、PNG/JPG 输出。
- 贴图：窗口创建、置顶、拖拽/缩放状态、关闭生命周期。
- OCR：语言选择、不可用提示、识别结果面板、复制文本。

手工验证清单：

- 单屏截图。
- 多屏截图。
- 不同 DPI 缩放。
- 副屏在主屏左侧或上方的负坐标布局。
- 跨屏框选。
- 截菜单、悬浮提示和右键菜单。
- 复制截图到聊天和文档。
- 保存 PNG 和 JPG。
- 贴图拖拽、缩放、关闭。
- 简中、繁中、英文和中英混排 OCR。
- OCR 能力缺失时的提示。

## 发布策略

虽然作为一个大版本发布，内部开发需要拆成可验证模块：

1. 截图快捷键、命令入口和生命周期。
2. 多屏 overlay 和选区。
3. 标注工具和输出管线。
4. 保存和剪贴板。
5. 贴图窗口。
6. 本地 OCR。
7. 设置页和最终集成。

开发包通过自动化测试和手工验证后，再打正式 release。
