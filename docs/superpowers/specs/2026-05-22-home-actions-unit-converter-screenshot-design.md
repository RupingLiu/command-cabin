# 首页单位换算与截图入口设计方案

## 背景

CommandCabin 现在的单位换算主要通过启动器输入框触发，例如输入 `1cm`、`1kg`、`1cm * 1cm * 1cm`。这个方式对熟悉命令的人很快，但对普通用户不直观。用户希望像百度单位换算一样，有明确的单位转换界面，并且首页直接提供“单位换算”和“截图”功能按钮。

截图功能已经具备选区截图、标注、文字、马赛克、保存、复制、OCR、置顶和快捷键设置。本轮不重写截图内核，重点是补一个明确入口，并把截图工具栏收口为更接近微信截图的浮动工具条体验。

## 目标

- 首页直接展示两个功能按钮：`单位换算`、`截图`。
- 点击 `单位换算` 进入独立单位转换页面，不要求用户记住输入命令。
- 单位转换页面只支持两个分类：重量、长度。
- 重量默认 `kg -> lb`，长度默认 `cm -> inch`。
- 单位选择通过下拉菜单完成，中间有双箭头按钮交换左右单位和值。
- 点击 `截图` 直接启动普通截图。
- 截图快捷键仍可在设置页配置。
- 截图工具栏更接近微信截图：围绕选区浮动，优先显示图标，文字作为 tooltip/无障碍标签。

## 非目标

- 本轮不增加面积、体积、速度等完整单位分类页面。
- 本轮不移除输入框里的快速换算能力；命令式换算继续保留。
- 本轮不做滚动截图。
- 本轮不新增延时截图首页按钮；延时截图仍可通过已有命令/快捷键能力使用。
- 本轮不重写截图采集流程，只调整入口和截图层 UI 行为。

## 首页入口设计

首页在搜索框下方增加一行固定功能按钮，只有启动器主页面显示：

- `单位换算`
  - 点击后切换到内置单位转换页面。
  - 不执行搜索，不污染搜索历史。
- `截图`
  - 点击后调用现有 `system.screenshot.capture` 命令。
  - 启动截图后隐藏启动器，进入截图层。

按钮使用小图标加文字，保持当前 8px 圆角、浅色/深色主题变量和焦点态。按钮不放进“最近应用”网格里，避免和用户固定应用混在一起。

## 单位转换页面设计

页面结构：

- 顶部：标题 `单位换算` 和 `返回` 按钮。
- 分类：两个分段按钮 `重量`、`长度`。
- 结果摘要：例如 `1 千克 = 2.20462 磅`。
- 转换区：
  - 左侧数值输入框 + 单位下拉菜单。
  - 中间双箭头交换按钮。
  - 右侧数值输入框 + 单位下拉菜单。

默认状态：

| 分类 | 左侧默认 | 右侧默认 |
| ---- | -------- | -------- |
| 重量 | kg       | lb       |
| 长度 | cm       | inch     |

单位列表：

| 分类 | 单位                |
| ---- | ------------------- |
| 重量 | kg、g、mg、lb、oz   |
| 长度 | cm、mm、m、inch、ft |

说明：用户提到长度里有 `mg`，这里按长度语境理解为 `mm`；`mg` 仍放在重量分类。

交互规则：

- 用户编辑左侧数值时，右侧实时换算。
- 用户编辑右侧数值时，左侧实时换算。
- 切换单位时，以用户最后编辑的一侧为基准重新计算另一侧。
- 点击双箭头时，左右单位和值互换。
- 空输入、非法输入、负号未完成输入等状态不报错，只清空另一侧结果。
- 数字格式沿用快速换算的简洁策略，优先显示 6 位有效数字，避免长尾小数。

## 截图入口和截图层设计

首页 `截图` 按钮调用现有截图命令：

- 命令 ID：`system.screenshot.capture`
- 系统命令：`screenshot.capture`
- 模式：普通截图，不延时

截图快捷键：

- 继续使用设置页现有 `截图快捷键`。
- 默认仍为 `Ctrl+Alt+A`。
- 本轮不新增第二套快捷键 UI。

截图层 UI 收口：

- 保留现有截图功能：矩形、椭圆、箭头、画笔、文字、马赛克、撤销、重做、OCR、置顶、保存、完成、取消。
- 工具栏从屏幕底部固定位置改为跟随选区浮动。
- 工具栏优先出现在选区下方；如果下方空间不足，放到选区上方；左右超出时自动夹到可视范围内。
- 工具按钮改为图标优先，文字进入 `title` 和 `aria-label`，让视觉更接近微信截图工具栏。
- 完成逻辑保持不变：点击完成、按 Enter、双击左键都会复制到剪贴板并关闭截图层。

## 架构方案

单位换算：

- 新增 `packages/core/src/unitConversion.ts` 存放纯换算逻辑、单位定义、格式化逻辑。
- Renderer 的单位转换页面只负责 UI 状态，不直接写换算公式。
- 以后如果要让输入框快速换算也复用这套逻辑，可以再逐步抽回 quick-converter 的重复公式。

首页入口：

- `App.tsx` 增加 `unit-converter` view。
- `LauncherPage.tsx` 增加首页功能按钮区域。
- `useLauncherController.ts` 增加直接执行截图系统命令的方法，避免页面组件直接依赖 IPC 细节。

截图：

- 复用 `screenshotController.start('capture')`。
- 调整 `ScreenshotOverlay.tsx` 中工具栏定位，让它基于选区和虚拟屏幕边界计算位置。
- 调整 CSS，使工具栏变成紧凑浮动条。

## 测试策略

- Core 单元测试覆盖重量和长度换算、交换、非法输入、格式化。
- Renderer 测试覆盖首页按钮出现、点击单位换算进入页面、点击截图调用命令。
- Renderer 测试覆盖单位转换页面默认状态、分类切换、单位选择、交换按钮、左右输入互算。
- Screenshot 测试覆盖工具栏定位：下方、上方、左右边界夹取。
- 运行 `corepack pnpm typecheck`、`corepack pnpm lint`、`corepack pnpm test`。

## 验收标准

- 打开软件首页即可看到 `单位换算` 和 `截图` 两个按钮。
- 点击 `单位换算` 后不用输入命令即可进行 kg/lb、cm/inch 等换算。
- 单位转换页面默认重量为 `kg -> lb`，默认长度为 `cm -> inch`。
- 双箭头按钮可以交换左右单位和值。
- 点击首页 `截图` 立即进入普通截图。
- 截图快捷键仍可设置并正常触发。
- 截图工具栏跟随选区显示，不再固定在屏幕底部。
